# LanHing-VirtualChat · 双 AI 对话模拟器

让**两个基于真实人设的 AI** 在一条连续流动的时间线上互相私聊——你导入双方的「人设档案」、设定他们当前的关系，然后看他们在某种可能里如何相处。你可以审核、肯定、否定、逐句纠正，让对话越来越像真人。

> 桌面应用，JavaFX + 任意 OpenAI 兼容服务（DeepSeek/OpenAI/Moonshot/Zhipu/Qwen/Gemini/OpenRouter/SiliconFlow…）。所有数据只存在本地，绝不上传。

## ✨ 特性

- **双 AI 私聊**：左右两套人设各跑一个 LLM 实例（任意 OpenAI 兼容服务），微信式聊天窗。
- **🕐 连续时间流逝表 + 可调流速**：顶部时钟匀速 N× 一直走（1× / 2× / 4× / 8× / 16× / 20× 可选），AI 自己决定**每句话在未来哪个时刻冒出来、能多久不回**——不强制一来一回，可以一个人连发、另一个半天不说；起始时刻可调（默认 21:00）。
- **💬 连发多气泡**：AI 输出的换行被自动拆成多个气泡，每条错开 5-25 模拟秒揭示，复刻真人「叮叮叮」连发感。
- **📷 AI 发表情包**：导入一个表情包文件夹给每一侧（独立），AI 按情境从库里选图发出来（一次 1-2 张）；给图打标签后越发越准。
- **🚶 离开事件触发**：AI 说要去做某事（画画 / 睡觉 / 打游戏 / 吃饭 …），系统自动插入旁白「X 去 Y 了」+ 按活动类型估算时间跳分钟（睡 7-10h / 画画 30-90min / 游戏 25-60min / …）+ X 回来后接话。
- **🤖 AI 开场**：点「AI 开场」让 AI 自己根据双方人设、关系和场景决定谁先开口、第一句说什么。
- **🗂️ 多时间线（迭代档案）**：随时新建/切换/重命名/删除时间线，每条时间线独立保存聊天历史、纠正、风格反馈和人设快照，互不污染——可以做"分支模拟"。
- **🎨 主题系统**：白天 / 黑夜 / 跟随系统切换 + 6 色预设（蓝/粉/绿/黄/红/紫）+ 任意自定义色。所有窗口和对话框统一风格。
- **内心 + 直出双栏**：左栏是发出去的话，右栏是那一刻的内心想法，悬停联动高亮。
- **导入即用、人人可用**：在软件里「导入左边/右边人设」，选两份蒸馏档案即可，不写死任何人。显示名自动跟着档案走。
- **当前关系可调**：「当前关系」栏手动设定两人此刻的关系/阶段（如"高中同学，刚表白第 1 天"），生成时注入，决定 AI 的亲密度与语气。
- **三种调教方式**：
  - 👍 **肯定** / 👎 **否定**整轮：弹框可选填一段说明（写了 → AI 按你说的整理；留空 → AI 自己看对话总结），以后自动往好的方向走、规避不像的聊法。
  - ✗ **逐句纠正**：某句不对，点 ✗ 写一句说明，该角色以后避免。
  - ✏️ **插话**：以任一角色或旁白手动插入一句。
- **📋 查看风格总结**：随时弹窗查看 AI 累计学到的「多保持」和「要规避」指引，可单独清空。
- **可选真实语料**：可配置一份导出的聊天记录，生成时按话题检索真实片段注入，让语气更贴近真人（不配置也能正常跑）。
- **自动存盘**：时间线、关系、纠正、风格指引、主题偏好全部本地自动保存，下次接着聊。

## 🚀 快速开始（普通用户）

1. 到 [Releases](../../releases) 下载最新版（任选其一）：
   - 📦 **`LanHing-VirtualChat-x.x.x-setup.exe`** — 一键安装包（推荐，含桌面快捷方式 / 卸载）
   - 📁 `LanHing-VirtualChat-windows.zip` — 绿色版，解压即用
2. 安装/解压后双击 `LanHing-VirtualChat.exe` 运行（已内置运行环境，**无需安装 Java**）。
3. 顶栏填入你的 **DeepSeek API Key** → 点「保存」。
   （去 https://platform.deepseek.com 申请，按量付费很便宜。）
4. 准备两份人设档案 → 见下方「完整教程」。

## 📖 完整使用教程

### 第 0 步　拿到两份「人设档案」

人设档案是一份 Markdown（`.md`），描述一个人怎么说话、什么作息、什么性格。推荐用配套的 **模拟对话蒸馏 Skill**（Claude Code）从真实聊天记录自动蒸馏：

> 👉 https://github.com/lingyunalingyun/Simulated-Dialogue

它会从一份聊天记录里**同时分出双方两份档案**。你也可以手写一份，只要第一行是标题：

```markdown
# 小明（备注/代号）

> 一句话身份：23 岁，程序员，和对方是大学同学

## 说话风格
- 口头禅：`原句` — 什么时候说
- ……
```
（软件取**第一行 `# 标题`** 里"（"之前的部分作为显示名，例：`# 小明（…）` → 显示「小明」。）

### 第 1 步　配置 API Key
启动软件 → 顶栏「Key」填 DeepSeek API Key、「模型」默认 `deepseek-chat` → 点「保存」。

### 第 2 步　导入左右人设
点「**导入左边人设**」选一份 `.md`（聊天窗左侧那个人），「**导入右边人设**」选另一份。
导入会把档案**复制进当前时间线** `data/timelines/<时间线名>/personas/`（之后软件可改这份副本，不动你的原文件）。导入后按钮、名字会自动变成档案里的名字。

### 第 3 步　设定「当前关系」
顶部「当前关系」栏填两人此刻的关系/阶段，例如：
- `高中同学，刚互相表白，在一起第 1 天`
- `在一起 3 年的老情侣`
- `刚分手两周，还有点别扭`

AI 会严格按这个阶段的亲密度说话。随时可改。

### 第 4 步　开场（两种方式）
**手动开场**：「开场场景」框写一句旁白起点（可留空）、「起始时刻」选时段（默认 21:00）、「先说」选谁先开口 → 点「**开始**」。

**AI 开场**：填好场景后直接点「**AI 开场**」，让 AI 根据人设、关系和场景自己决定谁先开口、第一句说什么（自动生成 1-3 条气泡作为开场）。

### 第 5 步　生成对话
点「**生成10轮**」：时钟开始匀速流动，消息按 AI 决定的时刻一条条冒出来；发够这一轮自动停。
- 「继续1句」：只生成一句。
- 「停止」：随时叫停。
- 时钟栏右侧可切**流速**（1×-20×），运行中也能改。

### 第 6 步　调教（让它越来越像）
- 看到很像的一段 → 点「**👍肯定**」；不像的一段 → 点「**👎否定**」。弹框里可选填一句说明（如「黏黏的感觉很对」或「太正经了」），留空就让 AI 自己看着办。
- 「**📋查看总结**」随时看 AI 累计学到的风格指引。
- 某一句不对 → 点气泡旁的「**✗**」，写一句"哪里不对/该怎么说"。
- 想自己插一句 → 底部输入框写好，点「以{左} / 以{右} / 旁白」。

### （可选）第 6.5 步　给 AI 一份表情包
1. ⚙ 设置 → 表情包库 → 「📷 左/右边表情包：未导入，点击选文件夹…」 → 选一个装满 png/jpg/gif 的文件夹（微信导出的 emoji 文件夹直接用）。
2. 跑对话时，AI 想发表情会输出 `[图:标签]` 或 `[图:?]`，软件随机或按标签从库里挑一张。
3. 看到 AI 发的图很合适 → 点图旁的 `#` 按钮 → 勾选 / 新增标签（如「委屈」「撒娇」），保存。
4. 之后 AI 会在合适的情境优先选带这些标签的图，越调越准。
5. 不想要某张图被发了 → 点 `✗` 记一句说明，以后会避免在这种情境下发图。

### 第 7 步　多时间线迭代
顶部「时间线」一栏：
- **新建** — 复制当前人设到新文件夹，从空白聊天开始（适合做"如果他俩没分手"的分支模拟）。
- **切换** — 下拉切换，每条时间线独立保存。
- **重命名 / 删除** — 当前时间线不能删。

### 第 8 步　主题
右上角 🎨 按钮：模式（白天/黑夜/跟随系统）+ 色调（白蓝/白粉/白绿/白黄/白红/白紫/自定义）。

### （可选）启用真实语料
若你有导出的聊天记录（WeFlow/WeChatMsg 等的 JSONL），可让 AI 参考真实片段。编辑软件目录下 `data/config.json`：
```json
{
  "corpusPath": "D:/路径/聊天记录.jsonl",
  "corpusLeft": "对方的微信昵称",
  "corpusRight": "你的微信昵称"
}
```
留空则不启用，纯靠人设档案也能跑。

## 🔒 数据与隐私

- 所有数据只存在程序目录的 `data/`：API Key、各时间线下的人设副本、对话历史、纠正、风格指引、日志。
- `data/` 已被 `.gitignore`，**绝不进入仓库、绝不上传任何服务器**。
- 仅用于个人创作与回忆，请勿用于骚扰、跟踪或侵犯他人隐私。

### 数据布局
```
data/
├── config.json              # 全局：API Key、模型、流速、主题、当前时间线
├── log.txt                  # 运行日志（每次启动重置）
└── timelines/
    └── <时间线名>/
        ├── timeline.json    # 聊天历史
        ├── corrections_*.json   # 双方逐句纠正
        ├── style_prefer.txt     # 👍 多保持指引
        ├── style_avoid.txt      # 👎 要规避指引
        ├── meta.json        # 当前关系 + 起始时刻
        ├── personas/        # 双方人设副本（left.md/right.md + left/images/, right/images/ 表情包库 + images_meta.json）
        └── rounds/          # 每次反馈的存档
```

## 🛠️ 从源码运行 / 构建

需要 JDK 21。

```bash
mvn javafx:run          # 直接运行
```

打包成内置运行环境的 Windows 应用（需 JDK 21 自带的 jpackage）：

```bash
# 1. 收集依赖
mvn -o dependency:copy-dependencies -DoutputDirectory=target/libs
mvn -o package
# 2. 用 jpackage 生成 app-image（内置 JRE）
jpackage --type app-image --name LanHing-VirtualChat \
  --input target/libs --main-jar LanHing-VirtualChat-1.0-SNAPSHOT.jar \
  --main-class org.example.personachat.Launcher \
  --icon src/main/resources/icon.png \
  --dest target/dist
```
生成的 `target/dist/LanHing-VirtualChat/` 整个打包成 zip 即可发布。

### 代码结构
```
src/main/java/org/example/personachat/
  Launcher.java       入口
  App.java            界面 + 对话调度 + 连续时钟 + 反馈/纠正 + 多时间线 + 主题
  Persona.java        人设 → system prompt
  Conversation.java   对话记录 + 双视角 messages + 滑动窗口
  ChatCorpus.java     可选真实语料载入与按话题检索
  DeepSeekClient.java DeepSeek API 调用
  Log.java            运行日志
src/main/resources/
  style.css           主题系统（looked-up colors，按 mode × accent 组合）
  icon.png            应用图标 / 启动 logo
```
